## Affected Pages/Routes
- macOS main app goal list — verify created goals appear with title, daily action, completed count, remaining count, and recoverable validation errors.
- macOS add goal form — verify valid goals persist and invalid target completion days are rejected.
- macOS Widget small/medium/large families — verify visible goals follow active sort order and fit without clipping.

## Scope Split

This file contains both vertical-slice checks and later MVP checks. The first implementation should complete the vertical-slice checks only.

Vertical slice includes:
- Create a goal in the main app.
- Read active goals from the Widget through shared App Group storage.
- Complete today from the Widget through `CompleteGoalIntent`.
- Reflect the completed state in repository snapshots and Widget UI.
- Reject duplicate same-day completions at the repository layer.

Later MVP checks include:
- Manual drag sorting UI.
- Archived history browsing.
- Full dashboard history and completion-rate views.
- Import/export, sync, distribution, and release packaging.

## Key Interactions to Verify
- Create “Vocabulary / Study 30 minutes / 200 days” in the main app, then confirm it appears in Widget data.
- Click an incomplete goal in the Widget and confirm it completes today without opening the app.
- Click an already completed goal in the Widget and confirm it stays completed without creating a duplicate record.
- Trigger a stale Widget tap for a missing goal ID and confirm repository returns goalNotFound and AppIntent logs/no-ops.
- Later MVP: archive a goal in the app, then confirm it disappears from Widget data but remains available for historical review.
- Later MVP: trigger a stale Widget tap for an archived goal and confirm repository returns goalNotActive and AppIntent logs/no-ops.

## Edge Cases
- LocalDay boundary around midnight.
- Same instant represented in different time zones.
- Target completion days <= 0.
- Later MVP: target completion days cannot decrease below completed count.
- Missing/deleted goal ID from stale Widget timeline.
- Long goal title and daily action strings in app list and Widget rows.
- Widget limits: small = 1, medium = 3, large = 5 for vertical slice.
- App Group entitlement missing or container unavailable.
- Duplicate Widget/app writes for the same goal/day.

## Critical Paths
- Widget Button -> CompleteGoalIntent -> GoalRepository.completeToday -> SwiftData App Group store -> Widget timeline reload -> Widget shows completed state.
- Main app creates goal -> repository persists goal with sortOrder -> Widget timeline reads activeGoalsForWidget(limit:day).
- Main app list reads same repository state after Widget check-in and shows updated completed/remaining counts.

## Unit Tests Required
- LocalDay produces stable YYYY-MM-DD local keys.
- GoalRepository.createGoal accepts valid data and rejects invalid target completion days.
- GoalRepository.completeToday inserts one completion per day, is idempotent, and rejects missing goals.
- GoalRepository.activeGoalsForWidget excludes archived goals, orders by sortOrder, respects limit, and marks isCompletedToday.
- GoalRepository enforces uniqueness by goal/day even when completion is requested more than once.
- If `archivedAt` is included in the vertical-slice model, GoalRepository.completeToday rejects archived goals even before archive UI exists.
- Later MVP: GoalRepository update rules allow title/action edits, lock startDate after first completion, allow target increases, and reject target below completed count.

## App Group Spike Verification Required
- Confirm the app and Widget extension share the same App Group identifier.
- Confirm the SwiftData store URL resolves inside the App Group container.
- Create one goal in the app and read it from the Widget timeline provider.
- Run `CompleteGoalIntent.perform()` and confirm the repository reads back exactly one completion for today's `LocalDay`.
- Confirm Widget reload is requested after completion. Treat OS refresh timing as eventually consistent, not instantaneous.

## Manual Widget Verification Required
- Add small Widget, verify 1 goal visible and tappable.
- Add medium Widget, verify 3 goals visible and tappable.
- Add large Widget, verify 5 goals visible and tappable.
- Tap Widget goal and confirm visible completion state refreshes without opening the app.
